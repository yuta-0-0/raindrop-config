# Raindrop / Obsidian 自動棚卸しルール v3.0

## 0. 結論

月次棚卸しは、ユーザー確認を前提にしない。
AIがURL・title・excerpt・folder・既存tagsをもとに、必ず1つの最終案を出す。
不明なものも放置せず、決定木に従って `要確認` を付けたうえで暫定分類する。

このルールの目的は、毎回ユーザーがCSVを目視確認しなくても、Raindropのタグ・フォルダ・Obsidian候補が一定品質で整う状態を作ること。

---

## 1. 基本方針

- Raindropは一次倉庫。
- Makeは日常保存の仮処理。
- チャッピーは月次棚卸しの正本判断。
- Obsidianは選別済み判断資産の倉庫。
- ユーザー確認は必須工程にしない。
- AIは迷っても必ず暫定決定する。
- 迷った場合は `要確認` を付けるが、フォルダ・タグは空にしない。
- CSV再分類では、既存タグを正本にせず、分類材料として扱う。
- `要確認` タグがついていて、かつAIの推奨フォルダと現在のフォルダが異なる場合は、現在のフォルダを維持する（ユーザーが意図的に移動したと見なす）。
- `要確認` タグが外れていてフォルダが変わっている場合も、現在のフォルダを維持する（ユーザーが確認済みと見なす）。

---

## 2. 月次棚卸しの入力

基本入力：

- Raindrop CSV
- 現行フォルダ一覧
- 許可タグリスト
- この分類ルール

CSV列の優先度：

1. url
2. title
3. excerpt
4. note
5. folder
6. existing tags

本文取得できる場合はURL本文を優先する。
本文取得できない場合は、title / excerpt / url / folder / existing tags だけで暫定分類する。

---

## 3. URL確認ルール

### 3.1 原則

分類時はURL本文確認を優先する。
ただし、全URLの本文確認ができない場合でも処理を止めない。

### 3.2 URL本文確認できた場合

本文内容を最優先して分類する。
既存folderやtagsと本文が矛盾する場合は、本文を優先して修正する。

### 3.3 URL本文確認できない場合

以下の順で判定する。

1. titleに明確な商品名・サービス名・技術名があるか
2. excerptに用途・ジャンル・素材・対象があるか
3. urlドメインから業種や用途が推測できるか
4. 既存folderが大きく外れていないか
5. 既存tagsに検索価値のある名詞があるか

それでも曖昧な場合は、既存folderを維持し、タグに `要確認` を追加する。
ただしタグなしにはしない。

## 6. タグ付けルール

- タグは通常2〜4個。
- 最大5個。
- 最低1個は必ず付ける。
- 1件につき、可能ならAタグを1個付ける。
- Bタグは検索の決め手になる具体名を優先する。
- Cタグは必要な時だけ付ける。
- `要確認` が付いていても、他のタグを必ず併用する。
- フォルダ名と同じタグは原則禁止。
- 感想タグは禁止。
- AIは許可タグ外を作らない。

## 9. Note欄ルール

Note欄にはAI判定理由を書く。
ただし、全件に長文は書かない。

形式：

`[判定理由] 1〜2文。分類根拠と迷いがあればその理由を書く。`

URL本文確認できなかった場合：

`[判定理由] URL本文未確認。title/excerpt/urlから〇〇と判断し、暫定で△△に分類。`

迷いがある場合：

`[判定理由] 〇〇要素と△△要素があるが、主用途が□□のため××に分類。必要なら要確認。`

## 10. confidence / 要確認ルール

月次棚卸しでは、内部的に confidence を持つ。
ただし、ユーザー確認を必須にしない。

### High

本文またはexcerptで用途が明確。
そのまま反映候補。

### Medium

title / excerpt / url から十分に推定できる。
そのまま反映候補。

### Low

本文不足・情報不足・複数カテゴリ混在。
`要確認` を付けるが、暫定フォルダ・暫定タグは必ず決める。

## 11. Obsidian昇格ルール

### Obsidian候補を付ける条件

以下のどれかに該当する場合、`Obsidian候補` を付ける。

- 制作や開発で再利用できる
- PRISM.blue / MIME.blue / ポートフォリオ等のプロジェクトに接続できる
- デザイン判断の根拠になる
- 実装判断の根拠になる
- 複数回参照しそう
- 比較・検討・購入判断の記録として残す価値がある
- 自分の言葉でメモを足せる

### Obsidian候補を付けない条件

- ただのログインページ
- ただのショップトップ
- 一時的な購入候補
- ニュース
- 読み返す目的が薄いもの
- 情報が薄い商品ページ

### 強制的にObsidian候補にしやすい領域

- 実装・コード
- Webデザイン・参考
- 素材・フォント
- DIY・ものづくりの技術資料
- PRISM / MIME / ポートフォリオに関係するもの

## 12. 出力ルール

月次棚卸しでは以下を出す。

1. Raindropインポート候補CSV
2. レビュー用CSV
3. 棚卸しレポート
4. タグ変更サマリー
5. フォルダ移動サマリー
6. Obsidian候補リスト
7. 低信頼だが暫定決定したリスト

ただし、運用の主成果物は `Raindropインポート候補CSV` とする。
レビュー用CSVは補助であり、ユーザー確認を必須にしない。

---

## 13. 最終判断基準

優先順位：

1. 後から検索できること
2. フォルダとタグの責務が分かれていること
3. 迷っても空欄にしないこと
4. 許可タグ外を増やさないこと
5. 月1で続けられること
6. Obsidianをノイズ倉庫にしないこと

迷った場合の最終ルール：

- フォルダは必ず1つ決める
- タグは必ず1〜5個決める
- 不確実なら `要確認` を付ける
- それでも処理は止めない
